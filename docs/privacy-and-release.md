# 隐私、安全与发布方式

## 隐私边界

Shadow 的目标不是把所有内容搬到云上，而是在保证本地体验的前提下，只把真正需要的最小公共数据送到云端。

## unknown 贡献默认关闭

- 首次使用时默认不授权
- 开启后只上传：
  - 歌名
  - 歌手数组
  - 专辑名
- 不上传：
  - 网易云 Cookie
  - 聊天记录原文
  - 好友 UID
  - 完整歌单
  - GPT 内容

## 基础安全策略

- unknown 提交接口限流
- 云端按标准化 key 去重
- 管理接口要求管理员令牌
- 日志避免记录敏感正文
- 客户端所有本地运行产物写回自身数据目录

## 发布方式

- 源码展示放在独立公开仓库
- Windows 便携包只放在 GitHub Releases
- 公开仓库不提交个人运行数据、数据库、日志、缓存与测试残留

## 发布前建议检查

- 便携包目录没有嵌套重复层级
- `data/` 只有空目录骨架
- 没有 `.db`、`.log`、`config.json` 等个人运行残留
- README 只面向普通用户，不夹带管理员流程
